## Agentic Quality Skills

When the user says `use agentic quality stack`, `use quality stack`, `use test driven development`, `verify red`, `lock tests`, `red flags`, `quality gate`, `golden benchmark`, `blind validation`, or `uplift loop`, route through the installed public skills:

- `test-driven-development`
- `agentic-quality-gates`
- `golden-benchmark-uplift-loop`

Use `docs/TRIGGER_DICTIONARY.ru-en.yaml` from the package as the expanded RU/EN trigger dictionary.

Default orchestration:

1. Behavior-changing code work → write/verify a failing test first.
2. Non-trivial or side-effecting work → classify risk and define acceptance gates before edits.
3. Prompt, skill, eval, scorer, or calibration improvement → use blind validation and measured before/after uplift.

Do not declare done from prose alone; report the proof command or artifact.
